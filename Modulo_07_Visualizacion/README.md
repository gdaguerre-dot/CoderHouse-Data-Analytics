# Proyectos del Módulo

## Cafeitaly Dashboard

Diseño conceptual de un dashboard operativo para una cadena de cafeterías orientado al monitoreo diario del negocio.

### Objetivos

- Monitorear ventas diarias.
- Evaluar el cumplimiento de metas comerciales.
- Identificar horarios de mayor actividad.
- Analizar productos con mejor desempeño.

### Diseño Aplicado

El dashboard fue construido siguiendo principios de jerarquía visual y experiencia de usuario.

#### Zona de Oro (Parte Superior)

KPIs estratégicos:

- Ventas Totales del Día.
- Meta de Ventas Diaria.
- Ticket Promedio.

Estos indicadores permiten evaluar rápidamente el estado general del negocio.

#### Zona Media

Visualizaciones analíticas:

- Gráfico de líneas para ventas por hora.
- Gráfico de barras horizontales para productos más vendidos.

#### Zona Inferior

- Tabla detallada de transacciones para análisis de detalle.

### Principios Aplicados

- Patrón de lectura en Z.
- Jerarquía visual.
- Separación entre resumen ejecutivo y detalle operativo.
- Diseño orientado a la toma de decisiones.
- Simplicidad y reducción de ruido visual.

## Data Coffee Storytelling

Desarrollo de una narrativa analítica completa orientada a explicar el desempeño comercial trimestral de una cadena de cafeterías.

### Modelo Narrativo Aplicado

**Contexto → Problema → Solución**

### Contexto

Indicadores principales:

- Ventas trimestrales: $160.000.
- Meta trimestral: $185.000.
- Cumplimiento: 86,5%.
- Ticket promedio: $9,50.

### Problema Detectado

El principal factor asociado al incumplimiento de la meta fue la caída de ventas durante los días de lluvia.

**Hallazgo:**

- Los días de lluvia redujeron aproximadamente un 15% las ventas.
- La ausencia de cobertura en la terraza limitó la capacidad operativa durante jornadas climáticas adversas.

### Insight Principal

La incorporación de pastelería artesanal incrementó el ticket promedio desde $7,00 hasta $9,50 por cliente.

Este crecimiento compensó parcialmente la caída de ingresos provocada por los días de lluvia, aunque no resultó suficiente para alcanzar la meta trimestral.

### Recomendación

Implementar una cobertura temporal o retráctil para la terraza y desarrollar promociones específicas para días de lluvia con el objetivo de reducir la pérdida de ventas asociada a factores climáticos.

### Aprendizaje

La actividad permitió comprender que los dashboards generan valor cuando transforman datos en decisiones concretas mediante una narrativa clara, comprensible y orientada a la acción.

## Aplicación al Análisis de Datos

Este módulo permitió comprender que la visualización constituye la etapa donde los datos adquieren verdadero valor para los usuarios finales.

Los conocimientos adquiridos permitieron integrar conceptos de análisis, diseño visual, experiencia de usuario y comunicación estratégica, desarrollando reportes capaces de transformar datos en información accionable para organizaciones y equipos de gestión.
